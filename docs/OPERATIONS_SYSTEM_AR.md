# نظام العمليات - WOS-M

## نظرة عامة

نظام العمليات هو وحدة متكاملة لمراقبة وإدارة بوت WOS-M. يوفر واجهة تحكم شاملة للمالك عبر لوحة Discord.

## المكونات

### 1. فحص الصحة (Health Check)
- فحص حالة قاعدة البيانات
- فحص حالة الطوابير
- فحص معدل التقييد
- فحص سجل التدقيق
- فحص APIs الخارجية

### 2. المقاييس (Metrics)
- استخدام الذاكرة
- استخدام CPU
- مدة التشغيل
- عدد المستخدمين النشطين

### 3. الحوادث (Incidents)
- تتبع الحوادث في الوقت الفعلي
- تصنيف حسب الخطورة
- إجراءات الإصلاح المقترحة

### 4. التنبيهات (Alerts)
- تنبيهات Discord للمالك
- تصنيف حسب الخطورة (INFO, WARNING, ERROR, CRITICAL)
- تفعيل/إلغاء التنبيهات

### 5. النسخ الاحتياطي (Backup)
- إنشاء نسخ احتياطية تلقائية
- التحقق من سلامة النسخ
- استبعاد الملفات الحساسة (.env, *.pyc, logs)
- الاحتفاظ بـ 14 يوم من النسخ

### 6. التراجع (Rollback)
- نقاط استعادة محددة
- يتطلب تأكيد صريح
- سجل كامل للإجراءات

### 7. الترقيات (Upgrades)
- فحص قبل الترقية
- ترقيات مرحلية
- rollback تلقائي عند الفشل
- يتطلب تأكيد صريح

### 8. الإصلاح الذاتي (Self-Healing)
- إصلاح تلقائي للمشاكل الشائعة
- سياسة محمية ضد الإجراءات الخطرة
- تسجيل جميع الإجراءات

## الأمان

- **ملكية**: كل العمليات تتطلب صلاحيات OWNER
- **التأكيد**: العمليات الخطرة تتطلب تأكيد صريح
- **الاستبعاد**: لا يتم نسخ ملفات .env أو secrets
- **التدقيق**: كل إجراء يتم تسجيله في سجل التدقيق
